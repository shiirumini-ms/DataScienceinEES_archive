## Full Instructions

We are on the theme of data wrangling. Often in science and particularly in the world of data science you will be asked to review and contribute to other peopleâ€™s code or to build on existing code and improve it. We have set up this assessment to replicate this process.

This is a repository with data and a functioning `R` script. We have purposely written this code to be inefficient and a bit hard to follow. It's an example of not particularly brilliant programming - though the script does currently work! Your challenge is to make the code better - more efficient, better structured, tidier, clearer, and to produce and save the more finalised data outputs, including saved data objects and figures. No need to add any datasets or improving the statistical analyses, instead stick with what we have provided and focus on making the code itself better. Bonus points if you also make the repo more organised, edit this readme to tailor to yourself etc!  

Finally, this will be peer assessed. Each of you are being asked to provide feedback on a fellow students work - the student whose work you will provide feedback on is listed in the table below. At the end we'll release a list of all the ways the code could've been improved, and you can see how well you did!
